## STEP 0 — 중복 조사 방지 체크 (v1.3 신규)

인터뷰 시작 전, Q1(조사 주제)을 먼저 받고 Notion DB에서 유사 조사 이력 검색:

```
잠깐, 조사 주제부터 먼저 알려주세요:
Q0. 조사 주제가 무엇인가요?
```

Q0 답변을 받으면 즉시 Notion 자료조사 DB(01bef8b196e84e57ac85cebe81735e33)에서 조사주제 컬럼을 검색.

**중복 발견 시 (키워드 70% 이상 겹치는 조사 이력 있을 때):**
```
[중복 확인]
━━━━━━━━━━━━━━━━━━━━━━━━
비슷한 조사가 있습니다!

- {이전 조사 주제} ({이전 조사 날짜})
  통과 자료: {건수}건 | 카테고리: {카테고리}

재조사 하시겠어요?
→ Y: 새로 조사 진행 (기존 결과는 유지)
→ N: 기존 결과 링크만 제공하고 종료
→ U: 기존 자료 업데이트 (새 자료만 추가, 중복 제거)
━━━━━━━━━━━━━━━━━━━━━━━━
```

**중복 없으면:** 바로 STEP 1 인터뷰 진행.

## STEP 1 — 인터뷰 (7문항, Q0 이미 받았으므로)

Q0을 이미 받았으니 나머지 7문항만 한번에 출력:

```
[자료조사 인터뷰 - 추가 정보]
━━━━━━━━━━━━━━━━━━━━━━━━━━━━

Q2. 이 조사의 목적이 무엇인가요?
    (직접 활용 / 콘텐츠 제작 / 업무 참고 / 의사결정)

Q3. 카테고리를 선택해주세요.
    → 부동산 / AI기술 / 법률·정책 / 게임 / 맛집·장소 / 시장분석 / 기타

Q4. 우선순위 방향은?
    → 최신성 우선 / 정확성 우선 / 균형 / 출처 지정

Q5. 참조할 출처나 사이트가 있나요?
    (없으면 "없음")

Q6. 조사 깊이를 선택해주세요.
    → 얕게(빠름) / 보통 / 깊게(상세)

Q7. 제외할 조건이 있나요?
    (없으면 "없음")

Q8. 추가 특이사항이 있나요?
    (없으면 "없음")

━━━━━━━━━━━━━━━━━━━━━━━━━━━━
```

## STEP 7 — 규칙 자동학습 (v1.3 신규) ⭐

조사 완료 후, 이번 조사에서 패턴을 분석하여 새 규칙 추천:

**분석 기준:**
1. 통과율이 높았던 출처 → 해당 카테고리 참조출처로 추가 추천
2. 탈락 주요 사유가 일관된 경우 → 필터링 규칙 추가 추천
3. 특정 키워드 패턴이 유효했던 경우 → 검색 전략 규칙 추천

**출력 형식:**
```
[규칙 자동학습 제안]
━━━━━━━━━━━━━━━━━━━━━━━━
이번 조사에서 학습된 패턴:

📌 제안 규칙 1:
  카테고리: {카테고리}
  규칙명: {규칙명}
  내용: {규칙 내용}
  근거: {이번 조사에서 발견한 패턴}

📌 제안 규칙 2: (있을 경우)
  ...

이 규칙을 조사규칙 DB에 등록할까요?
→ Y: 전부 등록
→ 1,3: 1번, 3번만 등록
→ N: 등록 안 함
━━━━━━━━━━━━━━━━━━━━━━━━
```

대표님이 Y 또는 번호 선택 시 → 조사규칙 DB(b24c9539d506487c9094c6a21a25d7bf)에 자동 저장:
- 추가방식: 자동학습
- 활성여부: 체크
- 적용횟수: 0
