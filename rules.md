## A. 하위원칙

핵심 원칙 6개(운영 지침 v3.0 섹션 2)의 세부 실행 규칙.

### A1. 파일명 규칙
- **모든 파일에 버전 포함**: `파일명_v1.md`, `haemilsia_v3.html`
- **다운로드 파일명 패턴**: 기존 프로젝트의 파일명 패턴을 먼저 확인하고 따를 것
- **인수인계 파일 형식**: `세션인수인계_YYYYMMDD_N차_v1.md`
- **SKILL.md는 그대로**: 스킬 정의 파일만 예외 (파일명 자체가 `SKILL.md`)

### A2. Notion 저장 규칙
- **저장 전 2단계 확인 필수**: (1) "저장할까요?" → (2) "어디에 저장할까요?"
- **명시적 허락 없이 바로 저장 금지**
- **DB 생성 전 중복 확인**: `notion-search`로 유사 DB 존재 여부 반드시 확인
- **페이지 업데이트 시**: child page 있으면 `replace_content` 대신 `update_content` + `old_str`/`new_str` 사용

### A3. 스킬 적용 규칙
- **스킬 확인 순서**: `skill-guide.md` → `skill-manager` 스킬 활용
- **추천 우선순위**: 1순위 기존 설치 스킬 → 2순위 GitHub 검색
- **설치 전 기존 패턴 확인**: 이전에 설치한 스킬(file-organizer, gstack 등)의 설치 패턴을 먼저 참고
- **설치 경로 확인**: CLAUDE.md와 skill-guide.md에 명시된 경로를 반드시 확인 후 설치
- **스킬 없을 때**: 업무 완료 후 "이 작업을 스킬로 만들까요?" 확인

### A4. 세션 루틴 규칙
- **시작 루틴 (반드시 순서대로)**:
  1. 운영 지침 v3.0 fetch (Notion: `3317f0809621816688feef408023224b`)
  2. 규칙 위반 DB 미해결 조회 (data_source: `27c13aa7-9e91-49d3-bb30-0e81b38189e4`)
  3. 보고 후 "오늘 뭘 할까요?" 질문
- **종료 루틴 (반드시 전부 실행)**:
  1. 작업기록 DB 저장
  2. 에러로그 기록 (에러 있을 때)
  3. 규칙 위반 기록 (위반 있을 때)
  4. 세션 인수인계 `.md` 파일 생성 → `/mnt/user-data/outputs/`
  5. Slack 알림 (C0AEM5EJ0ES)

### A5. 도구 추천 규칙
- **매 업무 시작 시 필수**: 기본값은 Claude Code
- **추천 형식**: "기본은 Code입니다. 이 작업은 **[도구명]**이 더 편합니다. (이유: ~)"
- **대표님 선택 존중**: "OK" 또는 "아니, [다른 도구]로 해줘"

### A6. 배포/설치 경로 규칙
- **Claude Code = 마스터**: push/배포 기본 담당
- **Antigravity = 백업**: Code가 못할 때만 사용
- **배포 전 검증**: 90% 룰 + preflight-check 적용
- **새 방법 시도 전**: 기존 작동하는 방법이 있는지 먼저 확인

## B. 자주 실수 패턴

미해결 규칙 위반 기록에서 추출한 반복 실수. 위반 발생 시 여기서 패턴을 확인하고 재발 방지.

### B1. 파일명 버전 누락 (빈도: ★★★)
- **증상**: 파일 생성 시 `_v1` 없이 출력
- **원인**: 파일 생성에 집중하다 네이밍 규칙 간과
- **방지법**: 파일 생성 직전 "파일명에 버전 있는가?" 자체 점검. 예외는 SKILL.md뿐

### B2. 세션 인수인계 미생성 (빈도: ★★★)
- **증상**: 세션 종료 시 인수인계 `.md` 파일을 만들지 않음
- **원인**: 대화 마무리에 급해서 종료 루틴 생략
- **방지법**: 대표님이 "마무리" 언급하면 즉시 종료 루틴 체크리스트 실행

### B3. 세션 시작 루틴 미실시 (빈도: ★★)
- **증상**: 운영 지침 fetch, 위반 DB 조회 없이 바로 업무 시작
- **원인**: 대표님의 첫 메시지에 바로 답하려는 성급함
- **방지법**: 첫 응답 전 반드시 2개 fetch 완료 → 보고 → 업무 시작

### B4. 도구 추천 미실시 (빈도: ★★)
- **증상**: 업무 시작 시 도구 추천 없이 바로 진행
- **원인**: 현재 도구(Claude.ai)에서 바로 처리 가능하다고 판단해 추천 생략
- **방지법**: 어떤 도구에서든 "이 작업은 [도구]가 적합합니다" 한 줄 필수

### B5. 스킬 설치 패턴/경로 오류 (빈도: ★★)
- **증상**: 기존 설치 패턴 무시, 잘못된 경로에 설치
- **원인**: 이전 스킬 설치 기록을 확인하지 않음
- **방지법**: 스킬 설치 시 `skill-guide.md` + 기존 스킬 폴더 구조 먼저 확인

### B6. Notion 저장 확인 누락 (빈도: ★)
- **증상**: "저장할까요?" 없이 바로 Notion에 기록
- **원인**: 자동화 습관으로 확인 절차 건너뜀
- **방지법**: Notion 쓰기 작업 전 항상 2단계 확인 (A2 참조)

### B7. 다운로드 파일명 패턴 미준수 (빈도: ★)
- **증상**: 기존 프로젝트 파일명 규칙과 다른 이름으로 출력
- **원인**: 프로젝트별 네이밍 컨벤션 미확인
- **방지법**: 파일 출력 전 기존 파일명 패턴 확인 → 동일 패턴 적용

### B8. 동기화 시 통합본 누락 (빈도: ★★★)
- **증상**: 개별 동기화 페이지(7개)만 수정하고, 상위 통합본(운영 지침 v4.0)에 반영하지 않음
- **원인**: system-docs-sync 스킬이 개별 페이지만 대상으로 관리, 통합본을 별도 체크하지 않음
- **방지법**: 동기화 시 반드시 **8개 대상** (개별 7개 + 통합본 1개) 전부 확인. 특히 CLAUDE.md 변경 시 통합본 필수 갱신

### B9. 스킬 설치 후 skill-guide.md 미등록 (빈도: ★★)
- **증상**: 새 스킬 설치 후 skill-guide.md에 등록하지 않아 추천 불가
- **원인**: 설치와 등록이 별도 프로세스로, 등록을 잊음
- **방지법**: 스킬 설치 완료 후 즉시 "skill-guide.md에 등록했는가?" 자체 점검. 로컬 + Notion 양쪽
