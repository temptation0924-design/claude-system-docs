# 노션 작업기록 규칙 (기본 규칙)

## 감지 키워드
작업완료, 세션종료, 오늘 작업

## 저장 위치
Notion 작업기록 DB

## 동작
- 작업 완료 / 세션 종료 시 → 작업기록 DB에 저장
- 다른 rules 파일에 해당하지 않는 일반 기록 요청도 이 규칙을 따른다 (기본 폴백)

## DB 자동 판단 기준
| 대화 내용 | 저장 DB |
|----------|---------|
| 오류/해결 | 에러로그 DB → error-handling.md 참조 |
| 작업완료/세션종료 | 작업기록 DB (이 파일) |
| 프롬프트 발견 | 프롬프트 라이브러리 DB |
| 프로젝트 변경 | 프로젝트 현황 DB |
| CLAUDE.md 수정 | 백업 페이지 |

## 기록 형식
| 필드 | 내용 |
|------|------|
| 작업명 | 수행한 작업 제목 |
| 작업 내용 | 상세 설명 |
| 결과 | 완료/진행중/보류 |
| 관련 프로젝트 | 해당 프로젝트명 |
| 날짜 | 작업일 |
| 다음세션인계 | 다음 세션에서 이어갈 내용 |

## 세션 시작 루틴
1. Notion 작업기록 DB 최근 3건 조회
2. 프로젝트 현황 확인
3. 미해결 에러 확인
4. 요약 보고

## 세션 종료 루틴
1. 작업 자동분류
2. "📝 작업기록 DB에 저장할까요?" 1회 확인
3. 승인 시 → Notion 작업기록 DB에 MCP로 직접 저장
4. 슬랙 완료 알림: ✅ [작업명] 완료 / 📌 내용: (요약) / 🔗 Notion: (링크)
5. 다음세션인계 컬럼 기록

## 주의사항
- 확인 방식: 1번만 질문, 임의 저장 절대 금지
- 민감정보(토큰, API키) 기록 금지
- 작업 완료 후 → 작업 진행 과정 상세 설명
- .md 마크다운으로 정리
