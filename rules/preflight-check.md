# 사전검증 (Preflight Check) 규칙

## 트리거 키워드
아래 키워드가 대표님 메시지에 포함되면 즉시 사전검증 프로세스를 실행한다:
- "검증해줘", "사전검증", "프리플라이트", "preflight"
- "설계서 검증", "코드 검증", "스크립트 검증"
- "배포 전 확인", "실행해도 되나", "문제 없는지 확인"
- "이거 돌려도 돼?", "이거 적용해도 돼?"
- "리뷰해줘", "점검해줘", "체크해줘"

## 검증 프로세스
1. Notion 에러로그 DB(a5f92e85220f43c2a7cb506d8c2d47fa) 조회 → 유사 에러 패턴 확인
2. Agent 1 (설계 검증): 로직·구조·변수·의존성 점검
3. Agent 2 (실행 검증): 파일경로·토큰·환경변수·에러패턴 교차확인
4. Agent 3 (엣지케이스): 특수문자·빈입력·타임아웃·stdin충돌·UX 점검
5. 종합 판정: CRITICAL 0건=PASS / CRITICAL 1건+=FAIL(수정 필수)

## 심각도 분류
- 🔴 CRITICAL: 실행 시 즉시 에러, 데이터 손실, 보안 위험 → 배포 차단
- 🟡 WARNING: 기능 작동하나 비효율/에러핸들링 부족 → 개선 권장
- 🟢 INFO: 스타일/주석/대안 존재 → 참고

## 2라운드 검증 시
- 이전 라운드와 다른 관점으로 검증
- NEW 이슈만 별도 표시
- 이전 이슈 반영 여부 확인
