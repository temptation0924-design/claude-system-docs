# 작업 단위 루틴 — 복습 카드 규칙

업데이트: 2026-04-11 | v1.0 신설

> **목적**: 대표님 능력 향상을 위한 **작업 단위 학습 시스템**. 하나의 작업이 끝날 때마다 복습 카드를 출력하여 **직후 복습 효과**(에빙하우스 망각곡선 — 하루 뒤 복습보다 15배 효과적)를 극대화한다.

---

## 트리거 조건

### 🤖 자동 트리거 (카드 무조건 출력)

다음 조건 중 **하나라도 해당**하면 작업 완료 직후 복습 카드 자동 출력:

| 조건 | 예시 |
|------|------|
| MODE 1 → 2 사이클 완료 | 기획 완료 + 실행 완료 (preflight 통과 → 승인 → 구현 끝) |
| MODE 3 검증 완료 | QA / 코드리뷰 / 배포 확인 사이클 완료 |
| **시스템 설정 변경** | settings.json, CLAUDE.md, session.md, rules.md, env-info.md, skill-guide.md 수정 |
| **파일 구조 변경** | 디렉토리 생성/이동, 새 파일 체계 도입 |
| **에러 해결 완료** | 디버깅 → 원인 분석 → 재발 방지 정리 끝났을 때 |
| **새 개념/패턴 도입** | 처음 도입하는 기술/시스템/워크플로우 (예: SessionStart 훅, DB 동적 조회) |

### 🙋 수동 트리거

| 대표님 명령 | 효과 |
|------------|------|
| "복습해줘" | 방금 한 작업 기준 복습 카드 생성 |
| "정리해줘" | 위와 동일 |
| "다시 설명해줘" | 이해 중심으로 재출력 (비유 강화) |
| "오늘 뭐 했지?" | 세션 통계 + 누적 복습 카드 요약 |

### 🤐 스킵 조건 (카드 만들지 않음)

- **한 줄 수정** (날짜 변경, 오타 교정 등)
- **단순 확인** (파일 내용 조회, 상태 체크)
- **반복 작업** (같은 패턴 여러 번 — 첫 회만 카드, 이후는 "[이전 카드] 참조" 한 줄)
- **애매한 경우**: 자동 출력 스킵, 대표님이 호출할 때만 제공 (**스팸보다 침묵**이 원칙)

## 원칙

1. **직후 복습 우선** — 작업 직후가 학습 효과 최대. 세션 끝까지 미루지 않기
2. **작은 작업은 스킵** — 신호 희석 방지. 중요한 것만 카드로 만들어야 진짜 기억됨
3. **비유 활용** — 대표님이 아는 도메인(부동산 운영, 일상 비즈니스)으로 설명하면 이해도 5배
4. **대표님 능력 향상이 최종 목표** — 장기 축적된 학습 카드가 대표님의 시스템 이해도를 단계적으로 끌어올림
5. **중복 감지** — 같은 개념 반복 도입 시 "[이전 카드 링크] 참조" 한 줄로 축약
6. **소요시간 계산** — `~/.claude/.session_start`의 epoch 활용 (SessionStart 훅이 기록)

## 세션 종료와의 관계

- **작업 단위 복습**: 하루에 여러 번 (task 단위) — **이 파일**에서 관리
- **세션 종료 루틴**: 하루에 한 번 — `session.md`에서 관리
- **중복 방지**: 세션 종료 시 복습은 하지 않음 (이미 작업 단위에서 처리됨). 대신 **세션 통계 요약**만 출력 (오늘 완료 작업 수 + 누적 카드 개수 + 소요시간)
