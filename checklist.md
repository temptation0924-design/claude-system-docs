## A. 공통 준비사항 (모든 업무 시작 전 필수 확인)

- [ ] **목표 확인** → 이 작업의 최종 결과물이 무엇인지 명확한가?
- [ ] **범위 확인** → 어디까지 하고, 어디서 멈추는지 정해졌는가?
- [ ] **기존 시스템 확인** → 이미 작동하는 것이 있는가? 있다면 건드려도 되는가?
- [ ] **영향 범위 파악** → 이 작업이 다른 시스템/파일에 영향을 주는가?
- [ ] **필요 도구/권한 확인** → API 키, MCP 연결, 서버 접근 등 준비됐는가?
- [ ] **70% 룰 적용** → 성공 확률 70% 이상인가? 미만이면 대안부터 제시
- [ ] **롤백 계획** → 실패 시 원상복구 방법이 있는가?
- [ ] **대표님 확인** → 계획을 공유하고 실행 승인을 받았는가?

## B. 업무 유형별 예상 어려움 & 대응

### 🔧 코드/배포 작업
| 예상 어려움 | 대응 방법 |
|------------|----------|
| 환경변수 누락 → 서버 에러 | 배포 전 `.env` 키 목록 대조 확인 |
| Git 충돌 → 배포 실패 | push 전 `git pull` 먼저, 충돌 시 수동 머지 |
| Railway 빌드 에러 | 로그 확인 → 70% 룰 적용 → 안 되면 대안 루트 |
| 기존 기능 깨짐 | 변경 전 백업, 단계별 테스트 후 배포 |

### 📝 문서/기획 작업
| 예상 어려움 | 대응 방법 |
|------------|----------|
| 목표/범위 불명확 → 재작업 | 시작 전 "최종 결과물 형태" 먼저 합의 |
| 정보 부족 → 진행 불가 | 필요한 자료 목록 먼저 정리 후 요청 |
| 포맷/구조 불일치 | 기존 문서 스타일 확인 후 통일 |

### 🔗 외부 연동 작업 (API, MCP, 웹훅 등)
| 예상 어려움 | 대응 방법 |
|------------|----------|
| API 키 만료/무효 | 연동 전 테스트 호출로 유효성 확인 |
| 외부 서비스 장애 | 대체 서비스 또는 수동 우회 방법 준비 |
| 권한 부족 → 접근 거부 | 필요 권한 목록 사전 확인, 대표님께 요청 |
| 응답 형식 변경 | 공식 문서 버전 확인, 에러 핸들링 추가 |

### 🆕 새로운 도구/기술 도입
| 예상 어려움 | 대응 방법 |
|------------|----------|
| 학습 비용 → 시간 초과 | 30분 내 기본 작동 안 되면 대안 검토 |
| 기존 시스템과 호환 안 됨 | 도입 전 호환성 체크리스트 작성 |
| 무료/유료 구분 불명확 | 비용 구조 먼저 확인 후 대표님께 보고 |
