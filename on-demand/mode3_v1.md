# MODE 3: 검증 모드 (Quality)

**트리거**: 배포 후 자동, "테스트해줘", "QA해줘", "배포 확인"

## 워크플로우

1. `/qa` — 자동 QA 테스트 (Playwright 기반 웹 + 버그 수정 루프)
2. `/review` — 코드 리뷰 (diff 기반, 구조적 이슈 탐지)
3. `/canary` — 배포 후 모니터링 (콘솔 에러, 성능 회귀, 스크린샷 diff)
4. `/cso` — 보안 감사 (필요 시, OWASP + STRIDE)
5. `/retro` — 프로젝트 완료 후 회고 (필수)

## 트리거 구분

| 컨텍스트 | "검증해줘" 의미 |
|---------|---------------|
| MODE 1 컨텍스트 | MODE 1 내 Preflight Gate (3 Agent 계획 사전검증) |
| MODE 3 컨텍스트 | 실행 후 코드 사후검증 (/qa + /review) |

## 모드 전환

- **검증 완료 → 운영**: 세션 종료 루틴 (session.md fetch)
- **검증 → 재실행**: 버그 발견 시 MODE 2 복귀 (묻지 말고 수정)

## 주의 (관련 규칙)

- **B15** 회고 누락: 프로젝트 완료 후 `/retro` 필수
- **B16** 보안 감사 스킵: 배포 전 민감한 변경은 `/cso` 필수
