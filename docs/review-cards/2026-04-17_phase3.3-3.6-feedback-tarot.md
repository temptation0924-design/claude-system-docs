## 카드 1 — 양방향 피드백 (positive + negative signal)

### 개념 한 줄
좋아요(상향 신호)와 싫어요(하향 신호)를 **둘 다** 받아야 진짜 취향을 학습한다.

### 부동산 비유 — 입주자 퇴실 인터뷰
해밀시아 운영에서 퇴실자에게 **불만만** 물어보면?
"주차 불편, 소음, 청소 상태…" — 불만 목록만 쌓임. 다음 호실은 "주차 넓고 조용하고 깨끗한 곳"이 됨.
하지만 **재계약자**에게 "왜 다시 사셨어요?" 물으면 진짜 답: "햇빛, 커피숍, 로비 분위기…" — **이게 진짜 매력 포인트**.

- 싫어요만 수집 → "싫지 않은 방"에 수렴 (불만 없는 밋밋한 공간)
- 좋아요도 수집 → "매력 있는 방"으로 성장 (사랑받는 특성 강화)

### 핵심 요약
- `blacklist_keywords` + `preferred_keywords` **두 테이블** 분리
- Haiku가 뉴스 제목에서 키워드 추출 → 양쪽 DB 적재
- Netflix/YouTube 철칙: **positive > negative** (긍정이 더 강력)
- 안전장치: `UNIQUE LOWER INDEX` + `INSERT OR IGNORE` + 500 cap + 90일 purge

## 카드 2 — Slack 3초 timeout 대응 (ack + background)

### 개념 한 줄
손님 응대는 **즉시**, 뒷작업은 **나중에** — 프런트와 백오피스 분리.

### 부동산 비유 — 해밀시아 프런트 데스크
손님이 "체크인 해주세요" 왔는데 윤실장이 "잠깐만요, DB 조회하고 키 만들고 청소 확인하고…" 3분 세워두면?
손님 짜증나서 **"됐어요, 다른 데 갈게요"** (= Slack timeout).

정석 운영:
1. **즉시 응대** (3초 안): "네, 접수됐습니다! 키는 5분 뒤 문자로"
2. **뒷작업 분리**: DB 조회·키 생성·청소 확인 → 백오피스 처리 후 푸시

### 핵심 요약
- Slack 버튼 클릭 → **3초 안 ack 필수** (안 그러면 timeout)
- Haiku 키워드 학습은 **background** 분리 (5~10초 소요)
- 구조: `handler()` → 즉시 ack → `_run_in_background(_bg)` → 완료 후 DB 저장
- 교훈: **무거운 작업 절대 동기 경로 금지**. 3초 룰 타협 불가.

## 카드 3 — MODE 2 Wave 전략 (4개 에이전트 병렬)

### 개념 한 줄
큰 공사를 4개 팀에 **동시에** 맡기되, 서로 간섭 안 하게 **구역**을 나눈다.

### 부동산 비유 — 해밀시아 대수선 공사
100평 리모델링 한 팀 순차 = **2주**. 4팀 병렬:
- 팀 A: 주방 · 팀 B: 화장실 · 팀 C: 거실 · 팀 D: 침실

**조건**: 공용 벽/배관 충돌 없어야 병렬 가능. 충돌(전기 분전반) → Wave 1 먼저, Wave 2에서 병렬.

### 핵심 요약
- 작업을 파일/모듈 **독립 구역** 나누기 → 충돌 없으면 병렬 안전
- `gsd-executor` 4명 동시 소환 → 시간 1/4로 압축 (실측)
- MODE 1 리뷰도 동일: CEO + ENG + Preflight **병렬** (독립 관점)
- 주의: **같은 파일 동시 수정 절대 금지** → Wave 쪼개기
