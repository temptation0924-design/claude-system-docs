🏷️ 트리거: 시스템 설정 변경 + 에러 해결 + 새 개념 도입 (3중 조건)

📌 이번 작업 한 줄 요약:
  각 규칙마다 "대표 계약서 1장(canonical page)"을 지정해두고 그 계약서의 재발 횟수만 런타임에 꺼내보는 체계로 전환하면서, 잘못 발부됐던 경고장(B19)을 회수했다.

🎯 핵심 개념 (대표님이 오늘 새로 배운 것):

  • **notion_page_id canonical mapping**
    — 해밀시아 102호에 계약서가 10장 쌓여 있어도, 공실률 리포트에는 "대표 계약서 1장"만 걸어두고 그 한 장의 재계약 횟수를 읽는 방식.
    — enforcement.json 각 규칙 → Notion DB 대표 페이지 ID 1개 고정 → slack-courier가 그 페이지만 찔러서 반복횟수 확인.

  • **미해결 + 최고 반복횟수 = canonical**
    — 여러 incident 중 "아직 안 끝났고 + 가장 자주 터진" 레코드가 대표.
    — 부동산 비유: 분쟁 많은 호실 중에서도 "현재 진행 중 + 민원 누적 최다"가 그 호실의 간판 사례.

  • **B19 오진 완전 삭제**
    — "sub-agent는 권한 상속 못 받는다"고 붙여놨던 경고장이 알고 보니 오진.
    — 진짜 원인은 B1 hook의 파일명 버전 차단이었고, bypassPermissions는 Anthropic 공식문서상 자동 상속됨.
    — 회수 절차: CLAUDE.md의 B19 v1 문구 제거 → v2로 재발행하며 "B1이 진범" 명시.

  • **MCP notion-fetch의 flat number 반환**
    — 원본 Notion API는 `{"number": 19}` 중첩 구조인데, MCP 래퍼는 `"반복횟수": 19` 평탄화해서 준다.
    — 부동산 서류 비유: 등기부등본 원본 대신 간이확인서를 받는 셈이라, `.number` 경로를 파면 null이 나옴. jq `.반복횟수`로 바로 접근.

🔄 과정 시각화:

  [Before]
  Slack 알림 → enforcement.json 반복횟수 필드 → 이모지 결정
  (매번 enforcement.json 수동 업데이트 필요 — 계약서 10장을 매번 다시 카운트)

  [After]
  Slack 알림 → enforcement.json의 notion_page_id 1개
           → MCP notion-fetch → "반복횟수" 평탄 값
           → 이모지/색상 체인 매핑 (1=💡 / 3=🟡 / 5=🟠 / 10+=🔴)
  (Notion이 single source of truth — 계약서 대표 1장만 보면 끝)

  [B19 회수 플로우]
  의심 → Anthropic 공식문서 재확인 → B1 hook 재현 테스트 9/9 PASS
       → CLAUDE.md B19 v1 삭제 → v2 재발행 ("B1이 진짜 원인")

✨ 잘된 부분:
  • canonical 선정 기준을 "미해결 + 최고 반복"으로 **2차원 조합**으로 잡은 것 — 단순 최신순/최다순보다 실무 신호가 정확함.
  • 오진이었던 B19를 덮지 않고 v2로 **공개 정정**한 점 — MEMORY.md에도 "재조사: B1이 진범" 기록 남김.
  • MCP 반환 구조를 실측하고 jq 경로를 고친 것 — 추측 디버깅이 아니라 실제 응답 샘플 보고 결정.

🔧 개선 가능한 부분:
  • canonical page가 **수동 지정**이라는 점 — 규칙 수가 늘면 휴먼 부담. 향후 "미해결+최다반복 auto-pick" 스크립트로 자동화 여지.
  • B19 같은 오진은 **공식 문서 먼저 확인**하는 습관이 있으면 처음부터 예방 가능. 앞으로 "~못 한다" 류 제약은 공식 문서 소스 필수 첨부.

💡 대표님이 응용할 수 있는 곳:
  • **해밀시아 호실별 민원 대시보드** — 호실마다 "대표 민원 1건" 지정 → 재발 횟수만 읽어서 등급 표시 (canonical mapping 동일 패턴).
  • **단기임대 분쟁 이력 관리** — 게스트별 "대표 분쟁 1건" 고정 → Slack 알림 색상 체인 그대로 재사용.
  • **AI 시스템 규칙 개정** — "~안 된다"고 써놨던 기존 규칙 중 실제 재현 테스트 없이 믿고 있던 것들 재검증 (B19 오진 교훈).
