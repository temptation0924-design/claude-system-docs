# 복습 카드 — B1 hook + 서브에이전트 권한 상속

## 한 줄 요약
메인 세션의 bypassPermissions는 서브에이전트에 자동 상속되며, hook 차단과 권한 거부는 에러 키워드로 구분한다.

---

## 주제 1. bypassPermissions 자동 상속 메커니즘

### 핵심 개념 (비유)
**본점 지점장 권한 자동 위임**
- 본점(메인 세션)이 "모든 출입 허용" 도장을 찍으면 → 지점(서브에이전트) 직원들도 **자동으로 같은 도장** 사용
- 지점마다 별도 발령장(`mode` 파라미터) 발급할 필요 없음
- 본점 도장이 **override 불가** — 지점에서 "저는 제한 모드로 할게요" 불가능

### Anthropic 공식 문구
> "this takes precedence and cannot be overridden"
> (메인 세션 설정이 우선하며 덮어쓸 수 없다)

### 과거 오진(誤診) 정정
- 이전: "서브에이전트 권한은 별도 설정 필요" → **오진**
- 현재: "메인 세션이 bypassPermissions면 자동 상속" → **정답**
- 확증: 오늘 재현 테스트 3종으로 확인

### 잘된 부분
- 오진을 재현 테스트로 검증 → 공식 문서와 일치 확인 → MEMORY 업데이트까지 연결

## 주제 2. hook 차단 vs 권한 거부 구분법

### 핵심 개념 (비유)
**건물 출입 거부 시 2단계 확인**
- 1단계: **경비원(hook)** 막고 있나? → 에러 메시지에 `[B1 차단]`, `[REF]` 있으면 경비원
- 2단계: **건물주(권한)** 거부인가? → 위 키워드 없고 permission/policy 언급이면 건물주

### 진단 순서 (필수 암기)
1. 에러 메시지 키워드 스캔 → `[B1 차단]` / `[REF]` 있나?
2. 있으면 → hook 차단 (파일명 버전 누락 등) → 우회/재분류로 해결
3. 없으면 → 권한 문제 → bypassPermissions 설정 확인

### 응용처
- 해밀시아 봇 에러 디버깅 시 → 에러 로그 키워드 먼저 확인 후 계층 판단
- 슬랙 점검 에러 → hook 아니면 Railway 권한 의심

### 잘된 부분
- 두 계층을 혼동하면 30분 낭비 → 키워드 기반 2초 판별법 확립

## 주제 3. B1 hook 구조 (파일명 버전 규칙)

### 핵심 개념 (비유)
**관리사무소 프리패스 존 신설**
- 관리사무소(`~/.claude/`) **루트층(1층)**: 방문 카드 없이 자유 출입 (3.5번 예외)
- 관리사무소 **지하층(하위 디렉토리)**: 기존 명단 그대로 — SYSTEM_FILES/숨김파일/EXEMPT_DIRS/CODE_EXTENSIONS 4단 화이트리스트
- 결과물(.md, .pdf, .docx)만 **방문 카드(_v1) 강제**

### 화이트리스트 4단 + 예외 (순서 중요)
1. SYSTEM_FILES 리스트 매칭
2. 숨김파일 (`.`으로 시작)
3. EXEMPT_DIRS 경로 매칭
3.5. **~/.claude/ 루트 레벨 자동 예외** ← 신규 추가
4. CODE_EXTENSIONS 확장자 매칭
→ 모두 탈락하고 DELIVERABLE_EXTENSIONS이면 버전 강제

### 응용처
- 새 결과물 파일 생성 시 → `_v1` 서픽스 자동 부여
- `~/.claude/` 루트 설정 파일 수정 시 → 버전 강제 없음 (CLAUDE.md, session.md 등)

### 잘된 부분
- 시스템 파일과 결과물 파일의 **목적 차이**를 디렉토리 위계로 반영 — 루트는 설정, 하위는 산출물

---

## 오늘의 교훈

1. **권한 상속은 자동** — 서브에이전트마다 `mode` 파라미터 추가하던 과거 습관 폐기
2. **에러는 키워드로 2초 분류** — hook vs 권한 혼동은 키워드 스캔 한 번으로 해결
3. **예외 규칙은 위계로** — 루트층(설정)과 하위층(산출물) 분리해 화이트리스트 단순화
