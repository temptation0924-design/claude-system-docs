🎯 핵심 개념 (오늘 새로 배운 것):

  • **cwd 기반 메모리 격리** — Claude Code는 "어느 사무실에서 출근했냐(cwd)"에 따라
    서로 다른 캐비닛(메모리 폴더)을 본다. 같은 직원(사용자)이라도 출근 위치가
    다르면 다른 캐비닛에 메모를 남기게 된다.
    → **해결책: symlink** = "두 캐비닛 문에 같은 자물쇠를 다는 것".
       OS 레벨에서 inode가 동일해지면 어디서 출근하든 같은 캐비닛.

  • **MCP 버그 사전 차단 패턴** — 임차인이 입주 전에 미리 알려준 "특정 시간대엔
    엘리베이터 멈춤" 정보처럼, **호출 전에 재현 조건을 감지**해서 즉시 우회 경로로
    돌리는 방식. 시행착오 0회 = 입주 후 컴플레인 0건.

  • **임시빌드 byte-equal 정합성 검증** — 건물 도면이 현장과 같은지 검증할 때,
    복잡한 측량(파싱) 대신 **"같은 도면으로 다시 지어서 동일한지"** 비교.
    awk 섹션 분리 같은 잔재주는 false positive(허위 경보)를 만든다.
    → 가장 견고한 방법은 **재생산 후 1:1 대조**.

  • **외부 도구 버그 처리 3단계 워크플로우**:
    1) 진단 정밀화 (정확히 어느 조건에서 터지는지 재현)
    2) 우회 매뉴얼화 (운영 매뉴얼에 박아 넣어 재발 시 즉시 대응)
    3) 공식 신고 (본사에 이슈 제출 → 근본 해결 위임)
    → "수리공 부르되, 그 사이엔 임시 우회로 운영 멈추지 않게."

🔄 과정 시각화:

  [정합성 검증 — Before / After]
  Before: detector → awk 섹션 분리 → 비교  ❌ false positive
  After : 임시빌드 → byte-equal diff       ✅ 100% 정확

  [메모리 통합 — Before / After]
  Before: cwd-A → 캐비닛-A,  cwd-B → 캐비닛-B  (메모 흩어짐)
  After : cwd-A → 캐비닛-A,  cwd-B ──symlink──→ 캐비닛-A  (단일 진실)

  [MCP 버그 처리 — 3단계 파이프]
  진단 정밀화 → 우회 매뉴얼화 → 공식 신고(#50260, #50261)
       ↓              ↓                  ↓
   재현 조건       즉시 대응         근본 해결(본사)

✨ 잘된 부분:
  • 도구 자체가 잘못 짖은 것(false positive)을 스킬 탓이 아닌 **검증 로직 결함**으로
    바로 인정 → 빌드-비교 방식으로 전환 (책임소재 빠른 판단)
  • cwd 격리를 "왜 이렇게 설계됐을까" 추론 후 **OS 레벨 해법(symlink)**으로 마무리
  • timeout 진단을 5→10s, 6→12s로 **정량 조정** (감이 아닌 측정 기반)
  • Anthropic에 이슈 2건 정식 제출 → 시스템 외부 의존성도 능동적으로 관리

🔧 개선 가능한 부분:
  • detector 도구를 자산화한 후 **회귀 테스트 케이스**가 아직 없음
    → 다음에 또 false positive가 나면 같은 진단을 처음부터 반복할 위험
  • symlink는 **백업/복원 시 함정**이 됨 (실파일 1개를 두 번 백업하지 않도록 주의)
  • MCP 버그 매뉴얼이 rules.md에 들어가 있는지, 별도 troubleshooting 문서로 빠졌는지
    위치를 명확히 해야 다음 세션에서 빠르게 재조회 가능
