## 📌 이번 작업 한 줄 요약

건물 공지판(Slack)을 통일 규격으로 리모델링하고, 그동안 "관리사무소 직원이 수기로 해야 했던" 서류철 칸 만들기(Notion DB 컬럼 추가)가 사실은 "중개 시스템에서 원격으로 가능했다"는 걸 발견한 날.

---

## 🎯 핵심 개념 (대표님이 오늘 새로 배운 것)

### 1. Notion MCP DDL — SQL 문법으로 DB 스키마 변경

- **도구**: `mcp__claude_ai_Notion__notion-update-data-source`
- **구문**: `ADD COLUMN "경고사항" RICH_TEXT` (SQL DDL 그대로)
- **부동산 비유**: 그동안 "대표님이 Notion 웹에서 직접 클릭해 속성 추가해주셔야 해요"라고 안내했던 것이, 사실은 중개사무소 본사에서 지점 서류철 칸 구성을 **원격으로 바꿀 권한을 이미 들고 있었던 것**. 손님(대표님)한테 서류철 들고 오라고 부탁했던 게 불필요했음.
- **파급**: Notion DB 구조 변경은 이제 **대표님 개입 0분** 자동화.

### 2. Slack 알림 3종 업그레이드 (경로 통일)

- **①  next_action 힌트**: 위반 경고마다 "이렇게 조치하세요" 자동 첨부 — 공지판에 "문의는 관리사무소 101호"를 붙여두는 격
- **②  빈도별 이모지**: 💡 1회 / ⚠️ 2-3회 / 🚨 4-9회 / 🔴 10회+ — 민원 누적 게시판에 "이번이 3번째" 빨간 표시 붙이는 격
- **③  Notion 백업**: 작업기록 DB `경고사항` 필드에 압축 포맷 저장 — Slack은 공지판(휘발성), Notion은 영구 장부
- **접근법**: **A(점진 개선형) + 옵션 β** 채택 — handoff-scribe/notion-writer 재사용, 새 에이전트 신설 없음

### 3. MODE 2 무중단 진행 원칙 (feedback 저장)

- **규칙**: 실행 중 대표님께 질문 금지. 수동 개입 필요 사항은 "할 일 리스트"에 누적만
- **예외 3종**: (1) hard_block, (2) spec 범위 밖 새 결정, (3) 명백한 충돌
- **부동산 비유**: 공사 중에 사장님께 "못 하나 더 박을까요?" 매번 전화 걸면 사장님이 다른 일을 못 함. 공사팀이 알아서 끝내고, 손댈 수 없는 것만 목록으로 전달.
- **메모리 파일**: `feedback_mode2_no_interrupt_v1.md`
