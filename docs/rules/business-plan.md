# 사업계획서 / 아이디어 검증 기록 규칙

## 감지 키워드
사업계획서, 아이디어 검증, 사업 아이디어

## 동작
- 사업 아이디어 관련 대화가 감지되면 → `/office-hours` 실행을 안내한다
- /office-hours는 구조화된 문제 정의, 전제 검증, 대안 탐색을 수행하는 스킬
- 아이디어를 즉시 기록하지 않고, 먼저 검증 프로세스를 거친다

## 절차
1. 대표님이 사업 아이디어/계획을 언급
2. Claude 팀장이 `/office-hours` 실행을 제안
3. /office-hours 완료 후 → 디자인 문서가 자동 생성됨
4. 필요 시 Notion 프로젝트 현황 DB에 저장 여부 1회 확인

## 주의사항
- 아이디어 단계에서 바로 코드 작성하지 않음
- 70% 룰 적용: 검증 없이 실행하지 않음
- 대표님 확인 없이 방향 결정하지 않음
