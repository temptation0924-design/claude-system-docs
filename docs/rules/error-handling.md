# 오류/에러 기록 규칙

## 감지 키워드
오류, 에러, 버그, 안됨

## 저장 위치
Notion 에러로그 DB

## 동작
- 오류 관련 대화가 감지되면 → 에러로그 DB에 저장
- 오류 발생 시 → 처음부터 task 체크리스트 만들어서 체크
- 체크리스트 아래 → 전체 작업 진행 시각화 설명
- 오류 해결 후 → 원인 분석 + 재발 방지 정리

## 기록 형식
| 필드 | 내용 |
|------|------|
| 오류명 | 에러 메시지 또는 증상 요약 |
| 발생 위치 | 파일명, 함수, 라인 번호 |
| 발생 원인 | 근본 원인 분석 |
| 해결 방법 | 실제 적용한 수정 내용 |
| 재발 방지 | 동일 오류 방지를 위한 조치 |
| 관련 프로젝트 | 해당 프로젝트명 |
| 날짜 | 발생일 |

## 절차
1. 오류 감지 → task 체크리스트 생성
2. 디버깅 진행 → 체크리스트 업데이트
3. 해결 완료 → 원인 분석 + 재발 방지 정리
4. "에러로그 DB에 저장할까요?" 1회 확인
5. 승인 시 → Notion 에러로그 DB에 MCP로 직접 저장
6. 슬랙 알림 전송 → [`rules/slack-worklog.md`](slack-worklog.md) 참조 (상세 작업일지 포맷 · `#general-mode`)
7. **복습 카드 자동 출력** → 에러 해결 완료는 `rules/task-routine.md`의 자동 트리거 조건. `#claude-study` 채널로 복습 카드 발송 (원인 분석 + 재발 방지 인사이트 포함)

## 주의사항
- 임의 저장 절대 금지, 1번만 확인
- 민감정보(토큰, API키) 기록 금지
- 해결 후 반드시 원인 분석까지 완료
- 복습 카드는 **"에러 해결 = 학습 기회"** 원칙 — 같은 에러 재발 시 이전 복습 카드 참조로 축약 가능
