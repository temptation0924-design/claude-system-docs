# 쉬운 설명 브리핑 (Easy Briefing)

## 배경

MODE 1 7번에 "📘 계획 이해 브리핑"(기획 승인 직전 1회 비유 설명)이 있지만:

1. **발동 지점 제한적**: MODE 1 기획 흐름에서만 발동 → MODE 2(실행)·MODE 3(검증)·MODE 4(운영) 시작 시 "지금 뭘 하는 거지?" 의문에 대응 불가
2. **범용성 부재**: 대화형 질문("이거 뭐야?")에도 대표님이 쉽게 이해할 설명 레이어가 표준화되지 않음
3. **수동 재설명 경로 없음**: 답변이 어려웠을 때 "쉽게 다시" 요청할 통일된 트리거 키워드 없음

## 목표

- 대표님이 새 요청을 줄 때마다 **착수 전 1회** 쉬운 설명 자동 발동 (복잡도 적응형)
- 설명이 어려웠을 때 수동 트리거로 재설명 경로 확보
- MODE 1 7번 풀버전 브리핑을 브리핑 레이어로 **이관·통합**

비목표:
- 매 tool call마다 설명 (노이즈)
- 연속 작업 micro-task마다 설명 (MODE 2 이미 기획에서 1회 설명 완료)
- 1분 이내 마이크로 요청까지 설명 (오타 수정·한 줄 변경 등)

## 결정 사항

| 항목 | 결정 | 근거 |
|------|------|------|
| 발동 범위 | 대표님 요청당 1회 + 수동 트리거 | 노이즈 최소화 + 이해 확보 양립 |
| 포맷 | 복잡도 적응형 (원라이너 / 3줄 / 풀버전) | 단순 요청까지 3줄 찍으면 어색 |
| Skip 조건 | 연속 작업 + 마이크로 요청(<1분) | 80% 요청은 조용히, 20% 착수 브리핑 |
| 대화형 질문 | **스킵하지 않음** — 원라이너로 찍고 답변 | "뭘해야할지 쉽게 설명" 본래 의도 유지 |
| 파일 위치 | `~/.claude/briefing.md` 신규 | 허브 패턴 (session.md·rules.md·skill-guide.md 동급) |
| MODE 1 통합 | 7번 풀버전 → briefing.md로 이관, MODE 1은 참조만 | 단일 원천(single source) |
| 수동 키워드 | 6종 확정 | 아래 "수동 트리거" 참조 |

## 아키텍처

### 발동 흐름

```
사용자 메시지 수신
  ↓
[1] Skip 판정
  ├─ 연속 작업 트리거 ("계속해", "진행해", "OK", "이어서") → 🔇 스킵
  └─ 마이크로 요청 (Claude 판단 <1분: Read 1회, 오타 수정, 한 줄 변경) → 🔇 스킵
  ↓ (skip 아님)
[2] 복잡도 판정
  ├─ 단순: 질문 / 1스텝 액션 → 원라이너
  ├─ 중간: 파일 여러개 / 스킬 호출 / 2-3스텝 → 3줄 템플릿
  └─ 복잡: MODE 1 기획 진입 → 풀버전 (MODE 1 7번 본문 재사용)
  ↓
[3] 브리핑 출력
  ↓
[4] 작업 착수
```

### 파일 구조

```
~/.claude/
├── CLAUDE.md        # 수정 — 섹션 2 라우팅 맵 +1줄, 섹션 3 끝에 "전역 브리핑 레이어" 1문단
├── briefing.md      # 신규 — 쉬운 설명 루틴 전담 (발동 로직 + 포맷 3종 + 수동 트리거)
├── session.md
├── rules.md
├── skill-guide.md
├── env-info.md
└── agent.md
```

## 포맷 상세

### 원라이너 (단순)

```
🎯 [작업 요약 1줄] — 부동산으로 치면 [비유].
```

예:
> 🎯 CLAUDE.md에 "쉬운 설명" 기능을 추가합니다 — 부동산으로 치면 계약 전 중개사가 "오늘 뭐 할지" 한 줄 브리핑하는 거예요.

### 3줄 템플릿 (중간)

```
🎯 뭘 할지: [1줄]
🏠 비유: [부동산/일상 비유 1줄]
⏱ 예상: [소요시간] · [영향 범위: 파일 N개 / DB / 배포 등]
```

예:
> 🎯 뭘 할지: briefing.md 신규 생성 + CLAUDE.md 섹션 2·3 수정
> 🏠 비유: 중개사무소에 "오늘 고객 응대 원칙" 표 하나 붙이고, 대표 매뉴얼에 "이 표 참조" 한 줄 추가
> ⏱ 예상: 15분 · 파일 2개 (신규 1 / 수정 1), 배포 없음

### 풀버전 (MODE 1 기획)

기존 MODE 1 7번 "📘 계획 이해 브리핑" 포맷 그대로 이관:
- 큰 그림 1줄 요약
- 핵심 개념 비유 설명
- 예상 결과물 + 소요시간 + 의존성 다이어그램
- "궁금한 거 있으세요?" 질문

## 수동 트리거 (D모드)

아래 키워드가 대표님 메시지에 포함되면 **직전 답변·현재 작업을 더 쉬운 비유로 재설명**:

- `"설명해줘"`
- `"쉽게 풀어줘"`
- `"쉽게 설명해줘"`
- `"비유로 설명"`
- `"무슨 말이야?"`
- `"다시 설명"`

수동 트리거 발동 시 포맷은 **무조건 3줄 템플릿 이상** (원라이너 금지) — 대표님이 이해 못 한 상황이므로 상세 필요.

## CLAUDE.md 수정 범위

### 섹션 2 라우팅 맵 — 1줄 추가

```markdown
| "설명해줘", "쉽게 풀어줘", "쉽게 설명해줘", "비유로 설명", "무슨 말이야?", "다시 설명" | `briefing.md` | 쉬운 설명 루틴 (수동 재설명) |
```

### 섹션 3 끝 — "전역 브리핑 레이어" 1문단 추가

```markdown
### 전역 브리핑 레이어 (Easy Briefing)

모든 MODE 공통 — 대표님 요청당 1회 착수 전 쉬운 설명 자동 발동. 복잡도 적응형 (원라이너 / 3줄 / 풀버전). 연속 작업·마이크로 요청은 스킵. 상세는 `briefing.md` 참조.

- MODE 1 기획 진입 시: 풀버전 (기존 7번 "📘 계획 이해 브리핑" 본문)
- MODE 2·3·4 새 요청: 원라이너 또는 3줄
- 수동 재설명 키워드 수신 시: 3줄 이상 재설명
```

### MODE 1 7번 수정

본문 삭제하고 참조로 전환:

```markdown
6. **Preflight Gate** (자동) — ...
7. **📘 계획 이해 브리핑** — Preflight PASS 직후 자동 실행 → `briefing.md` 풀버전 참조
8. 대표님 승인
```

## 구현 단계

1. `briefing.md` 신규 작성 (발동 로직 + 포맷 3종 + 수동 트리거 + 스킵 조건)
2. `CLAUDE.md` 섹션 2 라우팅 맵에 1줄 추가
3. `CLAUDE.md` 섹션 3 끝에 "전역 브리핑 레이어" 1문단 추가
4. `CLAUDE.md` MODE 1 7번을 참조로 축약
5. `build-integrated_v1.sh --push`로 GitHub 통합본 재빌드 (Claude.ai 동기화)
6. 검증: 다음 세션 시작 시 요청당 1회 브리핑 발동 확인

## 리스크 & 대응

| 리스크 | 대응 |
|-------|------|
| 복잡도 판정이 자의적 → 브리핑 누락/남발 | briefing.md에 판정 기준 명시 (단순=1스텝, 중간=2-3스텝, 복잡=MODE 1) |
| "대화형 질문까지 발동" → 노이즈 우려 | 대화형엔 원라이너만 (1줄, 거슬리지 않음) |
| MODE 1 7번 이관 중 누락 | 기존 본문 그대로 briefing.md에 복사, MODE 1은 참조만 |
| 수동 트리거 키워드 오탐 (예: 일반 대화 "무슨 말이야?") | briefing.md에 "현재 작업·직전 답변 맥락이 있을 때만 발동" 조건 명시 |

## 검증 기준

- [ ] 새 요청 "xxx 해줘" → 원라이너 또는 3줄 브리핑 자동 출력
- [ ] "계속해", "진행해" → 브리핑 없이 조용히 진행
- [ ] "오타 수정해줘" → 브리핑 없이 즉시 수정

- [ ] "쉽게 설명해줘" → 직전 답변 3줄 이상 재설명
