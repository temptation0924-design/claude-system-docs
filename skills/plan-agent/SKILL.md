# Plan Agent

대표님이 명령하면 계획을 자동 수립하는 팀 에이전트.

## 핵심 원칙
1. **계획과 실행 분리** — 이 스킬은 계획만 수립. 실행은 별도 세션
2. **검증 후 승인** — 대표님은 검증 통과된 계획만 본다
3. **시각화 필수** — 계획 완료 시 반드시 시각화로 세세하게 설명
4. **대표님 오버라이드** — 검증 점수 무관, 대표님이 "통과!" 하면 통과

## 실행 절차

### Step 1: 정보 수집
1. Notion 프로젝트 현황 DB 조회 (`91fd98db80304dafa5fb6fe795e16905`)
2. 작업기록 DB에서 관련 이력 검색 (`1b602782-2d30-422d-8816-c5f20bd89516`)
3. 이전 세션 인수인계 파일 확인
4. checklist.md에서 미완료 항목 확인
5. skill-guide.md에서 활용 가능한 스킬 확인

### Step 2: 플래너 — 계획 초안 작성
서브 플래너 3개가 각 영역 담당 (Claude Code 팀 에이전트로 병렬):
- **로드맵 서브플래너**: 전체 Phase 구조 + 의존성 분석 + 예상 시간
- **리소스 서브플래너**: 도구 배정 + 스킬 매칭 + 에이전트 역할 정의
- **리스크 서브플래너**: 실패 가능성 분석 + 대안 루트 + 70% 룰 체크

통합하여 계획 초안 v1 작성.

### Step 1.5: 검증 에이전트 — 90% 룰 적용
채점 루브릭 (100점 만점, 90점 이상 통과):

| 항목 | 배점 | 기준 |
|------|------|------|
| 목표 분해 | 20점 | MECE하게 쪼갰는가 |
| 의존성 분석 | 15점 | 순차/병렬 논리적인가 |
| 도구 배정 | 15점 | 최적 도구 선택했는가 |
| 리스크 + 대안 | 15점 | 실패 시 Plan B 있는가 |
| KPI / 성공 기준 | 15점 | 측정 가능한 목표 있는가 |
| 기존 시스템 호환 | 10점 | Notion/스킬/세션 루틴과 호환되는가 |
| 대표님 기대 부합 | 10점 | 결과물이 원하는 것인가 |

- **90점 이상 + 개별 항목 60% 이상** → 통과 → Step 3으로
- **90점 미만** → 반려 사유 명시 → Step 2로 (플래너가 수정 후 재검증)
- **대표님 오버라이드** → 점수 무관 통과

### Step 3: 대표님 승인
검증 통과된 계획을 **반드시 시각화**로 제시:
- 프로젝트 우선순위 (추천 + 진행률)
- 단계별 계획 (Phase, 도구, 예상 시간, 의존성)
- 검증 점수 + 통과 내역

대표님 선택: 승인 / 수정 요청 / 우선순위 변경

### Step 4: Notion 자동 기록
승인 후 자동 실행:
1. **프로젝트 현황 DB 업데이트** — 상태/현재단계/다음액션 갱신
2. **프로젝트 로드맵 페이지 생성** (신규) 또는 업데이트 (기존)
   - 프로젝트 현황 DB의 해당 항목 하위에 생성
   - Phase별 체크리스트 + 도구 배정 + 의존성 포함
3. **checklist.md 업데이트** — 다음 액션 반영
4. **세션 인수인계에 반영** — 다음 세션에서 자동 인지

## 계획 계층
```
프로젝트 로드맵 (상위)
  └── 주간 계획 (자동 추출)
       └── 일일 계획 (자동 추출)
            └── 세션 계획 (세션 시작 시 자동)
```

## 우선순위 결정
- 에이전트가 추천 → 대표님 승인
- 추천 기준: 마감 임박 > 매출 직결 > 진행률 낮은 것

## 범위
- 전체 사업 통합: 단기임대 + 쁘띠린 부동산 + AI 운영 시스템
