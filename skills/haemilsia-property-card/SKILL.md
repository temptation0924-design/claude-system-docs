# 🏢 haemilsia-property-card — 해밀시아 부동산 수익카드

해밀시아 부동산 업무에서 다가구·수익형 물건을 카톡으로 공유하거나 은행에 제출할 때 쓰는 모바일 친화 분석 카드를 생성한다.

**과거 실전 사례:**
- 2026-04-09: 남양리 2055-3 매매분석 (민트/다크골드 2테마, 4회 수정)
- 2026-04-16: 남양리 2027-10 대환대출 분석 (다크골드 → 화이트로즈 색상 변경, 건물사진 base64)

---

## 1. 사용 시나리오 (모드 분기)

### MODE A: 매매분석 (purchase)
신규 매수 시 수익률 검토용. 매매가/대출/보증금/월세 → **연 수익률** 산출.

**필수 입력:**
- 주소
- 매매가, 대출금액, 금리
- 호실별 (보증금 / 월세+관리비)
- 건물 사진 (선택, 있으면 base64 임베드)

**산출 항목:**
- 매매가 / 대출 / 금리 / 보증금 합계
- 전체건물수익(월세 합계) / 한달이자 / 비용공제수익
- 인수비용 = 매매가 - 대출 - 보증금
- **연 수익률 = 비용공제수익 × 12 ÷ 인수비용 × 100**

### MODE B: 대환대출 (refinance) ⭐ 은행제출용
기존 대출 + 추가대출로 일부 호실 보증금 반환 → 재임대로 월세 상승 → 수익 개선 시뮬레이션.

**필수 입력:**
- 주소
- 현재 대출금액, 금리
- 현재 호실별 (보증금 / 월세)
- 변동 호실 (어느 호실을 어떤 조건으로 재임대할지)
- 추가대출 금액

**산출 항목 (현재 vs 대환 후 비교):**
- 대출구조 변경 (5억 → 8억 형태)
- 추가대출 사용내역 (보증금 반환 항목별 명세)
- 호실현황 BEFORE/AFTER (변경 호실 하이라이트)
- 월세 합계, 대출이자, 순수익 비교 카드
- 요약: 추가대출, 월세증가, 이자증가, **순수익 개선**

### MODE 자동 판별
- "대환", "추가대출", "재임대", "은행제출" 키워드 → MODE B
- "매매", "매수", "수익률 분석" → MODE A
- 애매하면 한 줄로 묻기: "매매분석인가요, 대환대출 분석인가요?"

## 2. 디자인 시스템

### 테마 라이브러리
| 코드 | 이름 | 메인 컬러 | 사용 케이스 |
|------|------|----------|------------|
| A | 민트 | #3CD9A3 | 신축/밝은 분위기 |
| B | 다크골드 | #C9A84C / 배경 #0c1222 | 럭셔리/프리미엄 |
| C | 화이트로즈 | #A57971 / 배경 #fff | 은행제출/정중 |

**대표님 선호 패턴 (관찰):**
- 매매분석 → 민트(A) 또는 다크골드(B)
- 대환대출/은행제출 → **화이트로즈(C)** (정중하고 깔끔)
- 첫 시안은 다크골드로 만들고 → 화이트로즈로 색만 바꿔달라는 흐름이 흔함

### 공통 레이아웃
1. **히어로** (480 × 360px): 건물사진 + 주소 + specs (층수/세대수/타입)
2. **핵심 배너** (Mode B): "현재 -X만 → 대환 후 +Y만" 한 줄 비교
3. **본문 섹션들** (24px padding):
   - 대출구조 / 매매분석 표
   - 호실현황 표 (BEFORE/AFTER 두 개, Mode B만)
   - 수익 비교 카드 (Mode B) 또는 요약 카드 (Mode A)
4. **푸터**: "본 자료는 ~참고용..." + 주소 + 날짜

**폰트**: Pretendard (CDN: jsdelivr.net/gh/orioncactus/pretendard)
**컨테이너 max-width**: 480px (모바일 폭 고정)

## 3. 실행 워크플로우

### Step 1: 데이터 파싱 + 검증
사용자 입력에서 데이터 추출:
- 주소 → hero-title, footer
- 호실 데이터 → 보증금(원), 월세(원) 단위 변환 (만원 → 원: × 10000)
- 변동 호실 (Mode B) → BEFORE/AFTER 매핑

**계산:**
- 월 이자 = 대출 × 금리 ÷ 12
- 비용공제수익 = 월세합계 - 월이자
- (Mode A) 인수비용 = 매매가 - 대출 - 보증금합계
- (Mode A) 연수익률 = 비용공제수익 × 12 ÷ 인수비용 × 100

**음수 표시**: 비용공제수익이 음수면 빨간색(#C75A5A) "역마진" 표시.

### Step 2: 테마 결정
- 사용자가 명시하면 그 테마
- 안 했으면 추천: Mode A → 다크골드, Mode B → 화이트로즈

### Step 3: HTML 생성
- 템플릿: `templates/template_C_화이트로즈.html` 또는 `templates/template_B_다크골드.html`
- 데이터 치환 (Python f-string 또는 직접 Write)
- 파일명: `[주소키]_[모드]_[테마코드]_[테마명]_v1.html`
  - 예: `남양리2027-10_대환대출_C_화이트로즈_v1.html`
  - 매매분석은 `[주소키]_물건현황표_[코드]_[이름]_v1.html`

### Step 4: 건물사진 처리
사진 파일이 있으면:
1. `base64 -i [경로] | tr -d '\n'` → base64 문자열
2. HTML의 `<img src="...">` → `<img src="data:image/png;base64,...">`로 치환
3. v2로 저장 (v1은 외부 이미지 참조 버전 유지)

사진이 없으면:
- 그라디언트 폴백 사용 (테마 색상 기반)
- "사진 준비되면 v2로 박아드립니다" 안내

**채팅 업로드 이미지는 파일이 아님** — 사용자에게 다음 중 하나 요청:
1. 파일로 저장 후 경로 알려주기
2. `~/Downloads/building_[주소키].png`로 저장 (정확한 파일명 필수)

### Step 5: PNG 캡처 (카톡 전송용)
1. `python3 -m http.server 8765` 백그라운드 실행 (`~/Downloads`에서)
2. browse 스킬 사용:
   ```bash
   B=~/.claude/skills/gstack/browse/dist/browse
   $B viewport 480x900
   $B goto "http://localhost:8765/[URL인코딩된파일명]"
   $B reload  # viewport 적용 보장
   $B wait --networkidle
   $B screenshot ~/Downloads/[파일명].png
   ```
3. **검증**: `file [PNG경로]` 로 480×N 확인. 1280 폭 나오면 viewport 재설정 후 reload.
4. 서버 종료: `kill [PID]`

**예상 결과**: 480 × 1500~2200 px PNG, 약 300~400KB

### Step 6: 결과 보고
- HTML, PNG 두 파일 경로 제공
- `open -R [PNG경로]`로 Finder에서 노출
- 카톡 전송 방법 안내 (드래그)

## 4. 수정 요청 대응

대표님이 수정 요청 시 자주 발생하는 패턴:

| 요청 | 작업 |
|------|------|
| "색상 바꿔" + HEX 코드 | 새 테마로 v1 생성 (예: 화이트로즈) |
| "사진 넣어줘" | base64 임베드 → v2 |
| "주소/호실 틀렸어" | HTML 수정 → 같은 v 유지 (덮어쓰기) |
| "은행제출용으로" | Mode B로 전환, 화이트로즈 추천 |
| "더 깔끔하게" | padding 줄이기, 색 톤 다운 |
| "카톡으로 보내야해" | PNG 캡처 단계 진행 |

**파일 버전 규칙:**
- 같은 데이터 + 색상만 변경 → 새 테마 코드, v1
- 같은 테마 + 데이터 추가/이미지 임베드 → 같은 테마, v2/v3
- 데이터 자체 수정 → 같은 파일 덮어쓰기

## 5. 주의사항 (실전 학습)

1. **음수 수익 솔직히 표시**: 데이터 그대로 보여주기. 대표님은 전문가, 가공 X.
2. **파일명 확장자**: 사용자가 `.png`로 저장했는데 HTML이 `.jpg`로 찾으면 폴백 작동 → 매번 확장자 확인.
3. **한글 파일명 + URL 인코딩**: browse는 file:// 차단 → 반드시 HTTP 서버 + URL 인코딩.
4. **viewport 적용 시점**: `goto` 전에 viewport 설정해도 적용 안 될 때가 있음. `reload` 한 번 더 호출.
5. **macOS NFD 한글**: Finder에서 한글 파일명이 안 보인다는 보고가 들어오면 `open -R [경로]`로 Finder 노출.
6. **카톡 호환 PNG 크기**: 480×2200 이내 + 500KB 이내가 압축/축소 없이 깨끗하게 표시됨.
7. **base64 임베드 시 파일 크기**: 800KB 사진 → 1.1MB HTML. 카톡 텍스트 채팅에 첨부할 때 PNG가 더 가벼움.
