# 검증 시스템 (v2.0 추가)

> **점검 실행 완료 후 반드시 이 체크리스트를 순회한다. Stop 훅이 미실행 시 차단한다.**

## 검증 트리거
- 점검 실행의 모든 단계(DB쿼리→규칙판정→보고서등록→미해결추적→슬랙조합) 완료 후 자동 실행
- 수동 트리거: "검증해줘", "검증 실행", "체크리스트 돌려"

## A. DB 쿼리 (7항목 x 3점 = 21점)

| # | 체크 항목 | PASS 조건 |
|---|---------|----------|
| A1 | 임차인마스터 쿼리 성공 | 에러 없이 results 반환 (0건도 PASS) |
| A2 | 미납리스크 쿼리 성공 | 동일 |
| A3 | 이사예정관리 쿼리 성공 | 동일 |
| A4 | 공실검증 쿼리 성공 | 동일 |
| A5 | 아이리스공실 쿼리 성공 | 동일 |
| A6 | 퇴거정산서 쿼리 성공 | 동일 |
| A7 | 신규입주자DB 쿼리 성공 | 동일 |

**게이트**: A 1개라도 FAIL → B~E 채점 불가, 폴백 전략 실행 후 재시도

## B. 규칙 판정 (9항목 x 4점 = 36점)

| # | 체크 항목 | PASS 조건 |
|---|---------|----------|
| B1 | 임차인마스터 25분기 판정 실행 | 판정 결과 목록 존재 (정상+오류 합계 = 쿼리 건수, 템플릿 제외) |
| B2 | 미납리스크 대응단계별 분류 | 카운트 합계 = 쿼리 건수 |
| B3 | 미납 오늘 입금예정 식별 | 연체입금예정일=오늘 필터 실행됨 |
| B4 | 이사예정 9분기 판정 실행 | 판정 결과 목록 존재 (아이리스 역방향 cross-check 2건 포함: 추가연장X 미등록 + 진행중↔임대완료 모순) |
| B5 | 이사예정 아이리스 30일 규칙 체크 | 추가연장X + 만기일-오늘≤30일 + 아이리스미연결 필터 실행됨 |
| B6 | 공실검증 13분기 판정 실행 | 판정 결과 목록 존재 (임대완료 크로스체크 + 특별업무수행 광고정보 체크 포함) |
| B7 | 아이리스 17분기 판정 실행 | 아이리스상태👤 vs 공실검증 비교 + 임차인중복연결 + 유령 relation 3-way (사건종료/페이지삭제/쿼리누락 구분) + 일반임대가능 입력일자 체크 + 기간내퇴거 예외규칙 적용 + 이사예정 미등록(퇴거진행중) 역방향 감지 |
| B8 | 아이리스 엑셀비교 정방향 | 📋아이리스[엑셀] vs 아이리스상태👤 비교 + 문자열 정규화 적용 |
| B9 | 신규입주자 7규칙 판정 실행 | 판정 결과 목록 존재 (템플릿 제외) |

## C. 점검보고서 (5항목 x 4점 = 20점)

| # | 체크 항목 | PASS 조건 |
|---|---------|----------|
| C1 | 전송실패 복구 (2단계) 실행 | 처리상태=전송실패 조회 실행됨 (0건도 PASS) |
| C2 | 중복 필터링 실행 | 비교키 매칭으로 신규/기존 분류됨 |
| C3 | 신규 오류 점검보고서 등록 | 신규 오류 건수 = 등록 건수 (0건도 PASS) |
| C4 | 소스페이지URL 포함 | 등록된 모든 건에 URL 있음 |
| C5 | 중복 스킵 로그 표시 | 스킵 건 있으면 로그 출력됨 (0건이면 자동 PASS) |

## D. 미해결 추적 (5항목 x 3점 = 15점)

| # | 체크 항목 | PASS 조건 |
|---|---------|----------|
| D1 | 미완료 항목 조회 | ☑️처리완료=false AND 처리상태≠전송실패 쿼리 실행됨 |
| D2 | 경과일 계산 | 각 항목에 경과일(오늘-점검일) 산출됨 |
| D3 | 에스컬레이션 판정 | 7일/14일 임계값 체크 실행됨 |
| D4 | 긴급도 자동 승격 | 해당 건 있으면 승격 실행, 없으면 "해당없음"도 PASS |
| D5 | 경과일 내림차순 정렬 | 미해결 목록이 경과일 큰 순서로 정렬됨 |

## E. 슬랙 전송 (5항목 x 3점 = 15점)

| # | 체크 항목 | PASS 조건 |
|---|---------|----------|
| E1 | 7개 DB 섹션 모두 포함 | 임차인/미납/이사/공실/아이리스/퇴거/신규입주자 전부 있음 |
| E2 | 미해결 항목 섹션 포함 | 미해결 N건 섹션 있음 (0건이면 생략도 PASS) |
| E3 | 요약 라인 형식 준수 | "오류 N건 \| 미납 N건 \| ⚡오늘 입금예정 N건 \| 📌미해결 N건" |
| E4 | 점검보고서 링크 포함 | 하단에 DB 링크 URL 있음 |
| E5 | 슬랙 실제 전송 완료 | slack_send_message 호출 성공 |

## 스코어 계산

총점 = (A PASS수 x 3) + (B PASS수 x 4) + (C PASS수 x 4) + (D PASS수 x 3) + (E PASS수 x 3)
최대 = 21 + 36 + 20 + 15 + 15 = 107점

| 점수 | 판정 | 조치 |
|------|------|------|
| 102~107 | ✅ PASS | 점검 완료 인정 (95% of 107) |
| 85~101 | ❌ RETRY | FAIL 항목 자동 복구 → 재검증 (최대 2회) |
| 0~84 | 🚨 MANUAL | 수동 개입 요청 |

**게이트 규칙**:
1. A카테고리 1개라도 FAIL → B~E 채점 불가, 폴백 재시도 먼저
2. E5 예외: 슬랙 전송은 대표님 승인 후 → E5 제외 104점 이상이면 전송 승인 요청

## 자동 복구 전략

| FAIL 항목 | 복구 방법 |
|-----------|----------|
| A1~A7 (DB쿼리) | 뷰URL → search API → DB스키마 재확인 순서로 폴백 |
| B1 (24분기) | 쿼리 데이터 기반 규칙 순회 재실행 |
| B7 (크로스DB) | 아이리스+공실검증 데이터 매칭 실행 |
| B8 (엑셀비교) | 아이리스 데이터에서 두 필드 비교 실행 |
| C1~C5 (보고서) | 중복비교→등록→URL 파이프라인 일괄 실행 |
| D1~D5 (미해결) | 점검보고서 재조회→경과일→에스컬레이션 일괄 |
| E1~E4 (슬랙형식) | 메시지 재조합 |

## 재검증 루프

1차 검증 → 채점 → 102점 이상? → 완료
                    ↓ 미만
           FAIL 항목 자동 복구
                    ↓
2차 검증 → 재채점 → 102점 이상? → 완료
                    ↓ 미만
           2차 복구
                    ↓
3차 검증 → 재채점 → 102점 이상? → 완료
                    ↓ 미만
           수동 개입 요청 (FAIL 목록 제시)

## 복구 불가능 케이스
- Notion API 503/장애 → "Notion API 장애. 수동 점검 필요" 출력
- Slack API 장애 → 점검보고서만 등록 + 대표님께 직접 보고
- 2회 재검증 후에도 95점 미만 → FAIL 항목 목록 + 수동 개입 요청

## 검증 결과 출력 형식

점검 완료 후 아래 형식으로 검증 결과를 출력한다:

PASS 시:
```
━━━ 🔍 임대점검 검증 결과 ━━━

A. DB 쿼리:     21/21 (7/7 PASS)
B. 규칙 판정:   36/36 (9/9 PASS)
C. 점검보고서:  20/20 (5/5 PASS)
D. 미해결 추적: 15/15 (5/5 PASS)
E. 슬랙 전송:   12/15 (4/5 PASS) ← E5 전송 대기

━━━━━━━━━━━━━━━━━━━
검증 스코어: 104/107점 ✅ PASS
FAIL: E5 슬랙 전송 (대표님 승인 대기)
```

RETRY 시:
```
━━━ 🔍 임대점검 검증 결과 (1차) ━━━
... (FAIL 항목 표시) ...
검증 스코어: 87/107점 ❌ RETRY

━━━ 🔧 자동 복구 실행 ━━━
A1: 임차인마스터 → search API 폴백... ✅ 복구 성공
B1: 12규칙 재판정... ✅ 복구 성공

━━━ 🔍 임대점검 검증 결과 (2차) ━━━
...
검증 스코어: 104/107점 ✅ PASS
```
