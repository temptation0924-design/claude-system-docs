# 3단 분류 체계 (v2.0 필수)

모든 판정 결과는 **A / B / C** 중 하나로 분류하여 슬랙에 표시한다. 섞지 말 것.

| 코드 | 이름 | 정의 | 표기 |
|------|------|------|------|
| **A** | 입력 오류 | 데이터가 잘못/누락되어 시스템 정합성 문제 (즉시 수정 필요) | ❌ 오류 |
| **B** | 확인사항 | 데이터는 정상, 담당자 액션 필요 (체크박스·독촉·토글 등) | ✋ 확인요청 |
| **C** | 진행 추적 | 오류 아님, 일상 업무 진행 상황 (참고용) | 📊 현황 |

**분류 기준 예시**:
- A: 이름 미입력, 계약서 미연결, 퇴거정산서 미작성, relation 유령링크, **거주상태 빈값**
- B: 공실검증 확인 체크박스 미체크, 사건종료 토글 필요, 잔금 미완납 독촉, **미납 긴급대처(N일 경과 위험)**
- C: 미납 상습연체자·기간내퇴거·1~2차문자·입금대기중 (일일 독촉 워크플로우), 퇴거정산 진행중, 이사예정 사이클

**원칙**:
- C는 기본적으로 슬랙에서 **건수만** 표시 (상세 건 숨김) — 상세는 독립 대시보드
- A·B만 상세 나열
- 각 규칙 파일(rules/*.md)의 분기에 `category: A|B|C` 태그 필수

# 슬랙 알림 형식 (v2.0 3단 섹션)

```
🏢 해밀시아 임대점검 — YYYY년 MM월 DD일 (요일) [빡센점검 or 간편점검]

━━━ ❌ 입력 오류 (N건) — 즉시 수정 필요 ━━━
📕 임차인마스터 (N건)
  ❌ [건물_호수_이름] — [오류 내용] 📎<소스 URL|소스 페이지>
📙 이사예정관리 (N건)
  ❌ [건물_호수_이름] — [오류 내용]
... (0건이면 섹션 생략 or "✅ 없음")

━━━ ✋ 확인사항 (N건) — 담당자 액션 ━━━
🔴 긴급 (N건)
  ✋ [건물_호수_이름] — [액션] [담당자: 윤실장/대표님]
🟡 일반 (N건)
  ✋ [건물_호수_이름] — [액션] [담당자]
... (긴급도별 subgroup 가능)

━━━ 📊 진행 추적 (N건) — 참고 ━━━
  📛 미납리스크 상습연체자/기간내퇴거 등 22건 (독립 대시보드)
  📒 퇴거정산 진행중 11건 (파이프라인 관리)
... (건수만 요약, 상세는 생략)

━━━━━━━━━━━━━━━━━━━
A 입력오류 *N건* | B 확인사항 *N건* | C 진행추적 *N건*
📋 점검보고서: https://www.notion.so/a74d6ce07341401c88ff57e68063d6bf

(A·B가 0건이고 C만 있으면 "오늘 조치 필요 사항 없음" 메시지 + C 요약)
(A·B가 0건이면 헤더에 ✅ 표시)

━━━ ⚠️ 미납 확인 필요 ━━━

📛 미납리스크 (N건)
  🔴 긴급대처 N건 | ⚠️ 상습연체 N건
  🔵 입금대기 N건 | 🟡🟠 문자단계 N건 | ⬜ 기간내퇴거 N건
  ⚡ 오늘 입금예정: [이름들]

━━━ 💰 정산 진행중 ━━━

📚 퇴거정산서 — 진행중[입금대기] N건

━━━ 📌 미해결 항목 (N건) ━━━

⏰ [3일 경과] FLAT2_203 공실검증 미연결 — 담당: 이대리
⏰ [1일 경과] FLAT2_301 상태미입력 — 담당: 윤실장
🔴 [7일 경과] A동_404 계약서 미연결 — 담당: 윤실장 ← 에스컬레이션

━━━━━━━━━━━━━━━━━━━
오류 N건 | 미납 N건 | ⚡오늘 입금예정 N건 | 📌미해결 N건
```
