# 퇴거정산서

> **판정 원칙**: 단계별 결핍 오류 감지 (날짜 기한 무관, 단계 자체가 신호).
> **M2 마이그레이션**: `createdTime < 2026-04-15`인 레코드는 점검 제외 (소급 미적용).

## 업무 플로우 (담당자별)

```
퇴거일 → [정산대기중]       아이리스 퇴실점검·청소·파손체크
      → [정산서결제대기]    윤실장이 정산서 생성 → 대표님 보고 (정산서파일 업로드)
      → [정산서송부]        윤실장이 임차인에게 발송 (송부일 기록)
      → [진행중[입금대기]]  임차인이 계좌번호 회신 → 대표님 입금 (계좌번호 기재)
      → [정산완료]
```

## 판정 분기

### Branch 1 — 정산대기중

- **판정 제외** (아이리스 외부 작업 중, 내부 점검 대상 아님)

### Branch 2 — 정산서결제대기

- 결핍 조건: `정산서파일` IS EMPTY (files 배열 = [])
- 오류 메시지: `[{Name}] 윤실장: 정산서 파일 업로드 필요`
- 알림 대상: 윤실장

### Branch 3 — 정산서송부

- 결핍 조건: `송부일` IS NULL
- 오류 메시지: `[{Name}] 윤실장: 송부일 기록 필요`
- 알림 대상: 윤실장

### Branch 4 — 진행중[입금대기]

- **결핍 조건 (우선)**: `계좌번호` IS EMPTY or IS NULL
  - 오류 메시지: `[{Name}] 윤실장: 임차인 계좌번호 기재 필요`
  - 알림 대상: 윤실장
- **액션 조건 (계좌번호 있음)**: `계좌번호` IS NOT EMPTY
  - 알림 메시지: `[{Name}] 대표님: 입금 필요 ({계좌번호})`
  - 알림 대상: 대표님

### Branch 5 — 정산완료 / 미연락대기

- **판정 제외**

## 마이그레이션 (M2)

- **적용일**: 2026-04-15
- **기존 27건 (정산대기중 17 / 결제대기 6 / 송부 4)**: `createdTime < 2026-04-15` → 점검 제외
- **신규 생성 건만 필드 누락 오류로 잡힘**
- 기존 건 소급 입력 필요 시 별도 세션에서 수동 처리

## API 구현 노트

- `정산서파일` 필드: Notion API에서 files 배열 반환. `len(files) == 0` 으로 결핍 판정.
- `송부일` 필드: Notion API에서 date 객체 또는 None. `date is None` 으로 결핍 판정.
- `계좌번호` 필드: rich_text 배열. 빈 배열 또는 plain_text 빈 문자열로 결핍 판정.
- `createdTime`: Notion 페이지 메타. `datetime.fromisoformat` 후 `>= 2026-04-15` 컷오프.
