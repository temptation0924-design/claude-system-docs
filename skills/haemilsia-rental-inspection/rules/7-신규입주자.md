# 신규입주자DB — 7규칙 판정

Default view (잔금완납=false) 쿼리 → 진행중 건만 대상.
템플릿 레코드 제외: 12개 건물별 `{건물}_000_윤실장_신규`.

**판정에 필요한 필드**:
| 필드 | 소스 | API 접근 방식 |
|------|------|-------------|
| 계약자명 | 신규입주자 직접 | title 읽기 |
| 건물명 | 신규입주자 직접 | select 읽기 |
| 호수 | 신규입주자 직접 | text 읽기 |
| 임차인마스터 | 신규입주자 직접 | relation 빈값 체크 |
| 계약금완납 | 신규입주자 직접 | checkbox 읽기 |
| 잔금완납 | 신규입주자 직접 | checkbox 읽기 |
| 계약일 | 신규입주자 직접 | date 읽기 |
| 잔금일 | 신규입주자 직접 | date 읽기 |
| 비고 | 신규입주자 직접 | text 읽기 (v3.8 추가 — 삼삼엠투 예외 판정용) |

**판정 순서** (v3.8: 삼삼엠투 예외 추가):

~~~
1. 계약자명 빈값 → ❌ 이름 미입력
2. 건물명 빈값 → ❌ 건물 미입력
3. 호수 빈값 → ❌ 호수 미입력
4. 임차인마스터 relation 빈값 → ❌ 임차인마스터 미연결
5. 계약금완납=false AND 계약일 존재 AND 계약일 < 오늘 AND 비고에 "삼삼엠투" 미포함 → ⚠️ 계약금 미완납 확인
   ※ 비고에 "삼삼엠투" 포함 시 rule 5 스킵 (삼삼엠투는 계약금 단계 없이 잔금 일시불 정상 패턴)
6. 잔금완납=false AND 잔금일 존재 AND 잔금일 < 오늘 → ⚠️ 잔금 미완납 확인
7. 계약일 존재 AND 잔금일 존재 AND 계약일 > 잔금일 → ❌ 날짜 오류
8. 위 전부 해당 없음 → ✅ 정상
~~~

**🛡️ 삼삼엠투 예외 (v3.8, 2026-04-14 대표님 확정)**:
- 신규입주자DB의 `비고` 필드에 **"삼삼엠투"** 포함 건은 일반 임대와 결제 구조가 다름
  - 계약금 = 0원 (계약금 단계 없음)
  - 잔금 일시불 (전체 금액 잔금일 한번에 입금)
  - 보통 "특별업무수행" 분류
- 계약금완납=false여도 정상 → rule 5 (계약금 미완납) 발동 안 함
- 잔금 미완납(rule 6)은 그대로 적용
- 실제 사례: S_308_양지성 (계약일=2026-04-10, 잔금일=2026-04-20, 계약금=0, 잔금=1,063,700원)
