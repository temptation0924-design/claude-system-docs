# 공실검증 — 13분기 수식 기반 판정 (v3.3)

📊전체이력 뷰 (확인=false) 쿼리 → Python에서 직접 판정.

> **2026-04-14 변경**: `최초공실시작일` 필드 사용 보류. 퇴거지정 날짜 추적은 아이리스공실 DB의 `입력일자👤` 필드에서 이미 담당 중이므로 공실검증 DB는 중복 필드 안 씀. 규칙 8(특별업무수행_퇴실점검일오류) 제거, 14→13분기.

**판정에 필요한 필드**:
| 필드 | 타입 | 용도 |
|------|------|------|
| Name | title | 식별 |
| 건물👤 | select | 건물명 |
| 호수👤 | text | 호수 |
| 공실사유👤 | select | 공실 사유 |
| 임대가능여부👤 | select | 임대 가능 여부 |
| 광고구분👤 | select | 광고 구분 (연결방/미끼방 등) |
| 대분류👤 | select | 대분류 |
| 소분류👤 | select | 소분류 |
| 확인 | checkbox | 확인 여부 |
| 아이리스공실 | relation | 아이리스공실 연결 |
| url | url | 소스페이지 링크 |

**판정 순서**:

```
1. 건물👤 빈값 → 🚨건물미입력
2. 호수👤 빈값 → 🚨호수미입력
3. 공실사유👤 빈값 → 🚨공실사유미입력
4. 임대가능여부👤 빈값 → 🚨임대가능여부미입력
5. 아이리스공실 relation 빈값 → 🚨아이리스미연결
6. 공실사유 ≠ 공용실/비임대 AND ≠ 특별업무수행건 AND 대분류 빈값 → 🚨대분류미입력
7. 공실사유 ≠ 공용실/비임대 AND ≠ 특별업무수행건 AND 소분류 빈값 → 🚨소분류미입력
8. 공실사유=특별업무수행건 AND (광고구분 있음 OR 대분류 있음 OR 소분류 있음) → 🚨특별업무수행_광고정보삭제필요 (초단기세입자 입주 상태이므로 광고구분/대분류/소분류 전부 빈칸이어야 함)
9. (아이리스상태자동=특별업무수행건 OR 임대가능여부=특별업무수행건) AND 공실사유≠특별업무수행건 → 🚨특별업무수행_상태값불일치 (3개 컬럼[아이리스상태자동/임대가능여부/공실사유]이 전부 특별업무수행건으로 동기화되어야 함)
10. 임대가능여부=임대완료 AND 확인 미체크 → 🚨임대완료→확인미체크
11. 임대가능여부=임대완료 AND 아이리스상태≠임대완료 → 🚨임대완료↔아이리스불일치
12. 아이리스상태=임대완료 AND 임대가능여부≠임대완료 → 🚨아이리스만임대완료
13. 위 전부 해당 없음 → ✅정상
```
