# 이사예정관리 — 9분기 수식 기반 판정 (v3.4)

📊전체이력 뷰 (완료=false) 쿼리 → Python에서 직접 판정.

> **2026-04-14 변경 (v3.4)**: 이사예정 ↔ 아이리스 역방향 cross-check 2건 추가. 임차인마스터 단계에서 forward는 잡지만, 이사예정 자체에서 누락/모순을 못 잡던 사각지대 보강.
> - **case A** (rule 7): 추가연장X 등록했는데 아이리스 미등록 — 임차인 rule 8(✅정상:추가연장X)이 아이리스 무관하게 정상 처리하는 사각지대
> - **case D** (rule 8): 이사예정 진행 중인데 아이리스가 이미 임대완료 (새 임차인 입주됨) — 기존 임차인 이사예정이 안 닫힌 모순

**판정에 필요한 필드**:
| 필드 | 타입 | 용도 |
|------|------|------|
| Name | title | 식별 |
| 건물👤 | select | 건물명 |
| 거주상태👤 | select | 거주중/추가연장X 등 |
| 만기일👤 | date | 만기 날짜 |
| 이사예정일👤 | date | 이사 예정 날짜 |
| 완료 | checkbox | 완료 여부 |
| 6️⃣ 6]아이리스공실 | relation | 아이리스공실 연결 (cross-check용) |
| 📊[이사]아이리스상태자동 | rollup | 아이리스상태 자동 (모순 감지용) |
| url | url | 소스페이지 링크 |

**판정 순서**:

```
1. 건물👤 빈값 → 🚨건물미입력
2. 거주상태👤 빈값 → 🚨거주상태미입력
3. 만기일👤 빈값 → 🚨만기일미입력
4. 거주상태=추가연장X AND 이사예정일👤 빈값 → 🚨이사예정일미입력
5. 거주상태=완료 AND 완료 체크 안 됨 → 🚨완료→체크미완료
6. 거주상태=추가연장X AND 이사예정일 있음 AND 이사예정일 < 오늘 → 🚨퇴실점검필요(날짜경과)
7. 거주상태=추가연장X AND 아이리스공실 relation 빈값 AND **만기일 - 점검일 ≤ 30일** → 🚨아이리스미등록(추가연장X)  ← case A (v3.6 30일 조건 추가)
8. 거주상태=거주중/추가연장X AND 아이리스 relation 있음 AND 아이리스상태자동=임대완료 → 🚨모순:이사예정진행중↔아이리스임대완료  ← case D 신규
9. 위 전부 해당 없음 → ✅정상
```

**🚨 case A — 아이리스미등록(추가연장X) 판정 기준** (v3.6 업데이트):
- 임차인이 "추가연장X"를 통보함 → 퇴거 결정 → 아이리스공실에 퇴거지정 등록 의무
- 아이리스 relation이 비어있다 = 퇴거 절차 시작 안 됨 → 윤실장 확인 필요
- 임차인마스터 rule 8(✅정상:추가연장X)이 아이리스 무관하게 통과시키는 사각지대 보완
- **⏱️ 30일 예외 (v3.6, 2026-04-14 대표님 지정)**: 아이리스공실 등록은 **만기일 1개월 전부터** 해도 됨.
  - 만기일 - 점검일 > 30일 → 빈칸이 정상 (아직 등록 시기 아님, 규칙 적용 스킵)
  - 만기일 - 점검일 ≤ 30일 → 빈칸이면 🚨오류
  - 예: 만기일=2026-07-01, 점검일=2026-05-25 → 37일 남음 → 빈칸 정상
  - 예: 만기일=2026-07-01, 점검일=2026-06-10 → 21일 남음 → 빈칸이면 오류

**🚨 case D — 모순:이사예정진행중↔아이리스임대완료 판정 기준**:
- 이 임차인의 이사예정이 완료=false (진행 중)인데 → 아이리스 페이지 상태가 이미 "임대완료"
- 의미: 새 임차인이 들어왔는데 기존 임차인 이사예정 정리 누락 (잔존 데이터)
- 조치: 이사예정 완료 체크 또는 아이리스 임차인 relation 분리 확인

**Notion 수식 (이사예정관리 — 신규 2건 추가, v3.4)** — 배포 대상 필드: **`검증결과`** (formula 타입):

> 기존 `검증결과` formula 최상단에 다음 2개 분기를 추가 (대표님이 Notion에서 직접 병합):
> - 이미 우선순위 높은 분기들 뒤에 추가하되, "✅정상" 같은 catch-all 앞에 위치해야 함
> - rollup 필드명 `📊[이사]아이리스상태자동` 사용

```
// case A — 추가연장X인데 아이리스 미연결 (만기일 30일 이내일 때만 오류, v3.6)
prop("거주상태👤") == "추가연장X" and empty(prop("6️⃣ 6]아이리스공실")) and not empty(prop("만기일👤")) and dateBetween(prop("만기일👤"), now(), "days") <= 30, "🚨 아이리스미등록(추가연장X)",

// case D — 이사예정 진행 중인데 아이리스가 이미 임대완료 (모순)
(prop("거주상태👤") == "거주중" or prop("거주상태👤") == "추가연장X") and not empty(prop("6️⃣ 6]아이리스공실")) and prop("📊[이사]아이리스상태자동").format().contains("임대완료"), "🚨 모순:이사예정진행중↔아이리스임대완료",
```
