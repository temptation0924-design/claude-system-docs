## API 제약사항 (절대 잊지 말 것)

| 제약 | 설명 | 대응 |
|------|------|------|
| 수식 결과 미반환 | `검증결과`, `[DB 정합성 검증]` 등 formula 필드가 `formulaResult://` 참조로만 반환됨 | Python에서 규칙 직접 판정 |
| Rollup 필드 생략 | `📊아이리스상태자동`, `이사예정_거주상태` 등이 `<omitted />`로 반환됨 | 원본 DB를 직접 쿼리해서 크로스 비교 |

> **핵심 원칙: 오류 뷰에 의존하지 말고, 전체 데이터를 가져와서 직접 판정한다.**

## DB별 점검 규칙

### 1️⃣ 임차인마스터
전체 쿼리 (사건종료=false) → 12규칙을 원시 필드로 직접 판정.
rollup 필드(이사예정건수, 공실검증건수 등)가 omitted되므로, 위성 DB를 별도 쿼리하여 relation 건수를 직접 카운트.

### 2️⃣ 미납리스크
🚨[연체]확인필요 뷰 사용 가능 (checkbox + formula 복합 필터이나 작동 확인됨).
원시 필드로 충분:
- `대응단계👤` → 긴급대처/상습연체자/입금대기중 등 분류
- `누적미납액👤` → 금액 표시
- `연체입금예정일👤` → 오늘 날짜와 비교 → ⚡오늘 입금예정 표시
- `연체횟수👤` → 연체 횟수 표시

### 3️⃣ 이사예정관리
**오류 뷰 사용 금지** — formula 필터가 API에서 작동 안 함 (0건 반환).
📊전체이력 뷰 (완료=false) 쿼리 → Python에서 직접 체크:
- `만기일👤` < 오늘 + `거주상태👤` = 거주중 → ❌ 자동연장 처리 필요
- `거주상태👤` = 추가연장X + `이사예정일👤` ≤ 오늘 → ❌ 퇴실점검 필요

### 4️⃣ 공실검증
📊전체이력 뷰 (확인=false) 쿼리 → Python에서 직접 체크:
- `공실사유👤` 미입력 → ❌ 오류
  - **예외: `임대가능여부👤` = 특별업무수행건은 제외** (단기임대 입주 중이라 공실사유 불필요)
- `아이리스공실` relation 미연결 → ❌ 오류

### 6️⃣ 아이리스공실
**핵심: 크로스 DB 비교 필수**
📊전체확인 뷰 (확인=false) 쿼리 + 공실검증 DB도 함께 쿼리.
Python에서 직접 체크:
- `아이리스상태👤` vs 공실검증의 `임대가능여부👤` 비교 → 불일치 시 ❌ 상태불일치
- 퇴실점검필요(날짜경과) → 이사예정관리의 날짜와 비교
- 이대리: 기간내퇴거→일반임대 가능 → 상태 전환 누락
- 이대리: 퇴거날짜지정→기간내퇴거 → 상태 업데이트 필요

**오류가 아닌 것:**
- 임차인마스터 미연결 → 공실이면 임차인 없는 게 정상
- 퇴거날짜지정/기간내퇴거 상태에서 공실검증 미연결 → 아직 퇴거 전이라 정상

### 7️⃣ 퇴거정산서
전체 뷰 쿼리 → `퇴거상황` = `진행중[입금대기]`인 건만 출력.
정산완료, 정산대기중, 정산서결제대기, 정산서송부 등은 출력하지 않음.

  ❌ [건물_호수_이름] — [오류 내용]

  ❌ [건물_호수] — [오류 내용]

📚 퇴거정산서 — 진행중[입금대기] N건

━━━━━━━━━━━━━━━━━━━
